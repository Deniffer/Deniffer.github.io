vscode debug指南

鸽了这么久终于想起来重新做第二期视频了，这期视频我来讲述如何在vscode里面debug kernel的一些操作，主要分为下面几点

    1. 首先是关于2020年6s081 有些地方无法打断点的问题 参考知乎link

    2. 关于debug流程以及原理的理解(补基础系列)

      • RISC && CISC 未来我觉得应该是RISC的天下(指令集的优势)
      • gdbinit file (什么鬼东西) 可参考我的知乎文章
      • gdbtmp ~
      • Makefile的小知识mit6s081 Makefile解读
        一个项目工程构建版本的控制工具

    3. vscode 远程debug的操作 -. SSH到虚拟机或者远程机器 SSH简介 -. vscode 配置remote SSH(免密登陆设置)推荐阅读我的知乎关于免密登陆 类似传统的ssh vscode的remote ssh插件是借用本地的open ssh。ssh免密登陆/鉴于我们会频繁的登陆虚拟机以及用scp命令拷贝文件，所以我建议还是弄一个免密登陆 -. lauch.json的配置 (只能是查官方文档，边查边用) vscode 调用其他application的config file。 -. task.json的配置 与lauch.json的区别 -.

    4. example ls.c文件的解读 -. source file的解读 -. debug 展示

    5. vscode 与传统gdb的区别 其实vscode调用的也是gdb这个进程，只不过他在最原始的gdb的基础上加了一层端口转发，你可以认为他们两个的功能上是完全一致的，只不过传统gdb提供的是command line environment ，vscode是图形化的界面。这就引申到一个问题，大家都知道linux很棒，但是绝大多数人还是更喜欢windows，绝大部分就是因为图形化带来的直观性。

过程详解

  1. 下载vscode vscode是开源且免费的，我觉得是很强大的一个ide 用习惯之后很方便
  2. 下载remote ssh 插件
  3. 下载openssh (如果windows没有自带的话)我下载好的压缩包
  4. 配置环境变量(ssh)
  5. 查看虚拟机内网ip
  6. 尝试ssh到ubuntu20.04
  7. 用vscode 的remote ssh 连接ubuntu20.04 并配置免密登录
  8. 配置lauch/task.json (安装c/c++ debug插件)
  9. 开始第一次vscode debug的体验
  10. 唠一唠 vscode gdb 与传统gdb 优劣
  11. vscode remote ssh 的linux版本支持
  12. task.json下载
  13. lauch.json下载
