如何理解interrupt

最直观对于interrupt的理解就是说，当我硬件ready之后，我可以发送一个信号给cpu，说我已经准备好你让我干的活了，这里有你要的东西（比如是磁盘io读取数据），又或者是说，键盘被敲击了，你得发个信号给cpu说，老哥我现在需要一些关注，我有一些东西让你帮我处理一下，然后生成一个interrupt给cpu处理。类似的这些你都可以理解为interrupt，但是有个问题，就是interrupt的效率并不高。 后来才有了polling，epolling这些东西的出现（cpu主动轮询监听）

上一次trap系列更多的是将kernel如何处理到来的trap，如何从user mode陷入到 kernel mode；而这次的系列会focus在interrupt在hardware level的生成/路由/以及当你敲击键盘的时候，kernel是如何让他display到console上的。前置知识，看过我的trap系列，熟悉kernel处理trap的流程，了解一些supervisor mode的寄存器的用法，

Supervisor Mode需要用到的register/hardware

  1. PLIC(Platform-Level Interrupt Controller) ,负责(route)路由外围设备(peripheral device ,如键盘/鼠标/打印机等)的interrupt到CPU核

  2. CLINT(Core-Local Interrupter) ,负责定时器相关的中断。

  3. 与console相关的寄存器

  • LSR(Line Status Register)寄存器:用来指示输入的字节是否准备好被用户进程读取
  • RHR(Receive Holding Register)寄存器：用来放置可以被用户进程读取的字节。当RHR中的一个字节被读取时，UART硬件将其从内部的FIFO硬盘中删除，当FIFO中为空时，LSR寄存器被置0
  • THR(Transmit Holding Register)寄存器：当用户进程向THR写入一个字节时，UART将传输这个字节

  1. 中断相关的寄存器

    • SIE(Supervisor Interrupt Enable) 寄存器用来控制中断，其中有1bit是控制外部设备的中断（SEIE ,supervise external interrupt Enable ),1bit控制suffer interrupt(一个CPU向另外一个CPU发出中断)(SSIE)，1bit控制定时器中断(STIE,supervise Time Interrupt Enable)

    • SIP(supervisor interrupt pending)寄存器，可以观察这个寄存器来判断有哪些中断在pending,等待

    • SPPbit : privilege level is set to user mode if the SPP bit is 0, or supervisor mode if the SPP bit is 1;

    • SIE bit : The SIE bit enables or disables all interrupts in supervisor mode. When SIE is clear, interrupts

      are not taken while in supervisor mode.
