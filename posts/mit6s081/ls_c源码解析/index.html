本文是在写lab one实现find这个操作的时候由于要仔细分析ls.c这个源文件，所以特意记录下来的一个源码解析，旨在给一些对源码看的不是特别懂的同学一些我的看法建议。（目前全网最详细！！！）。 分析过程遵循源码执行过程 从main开始，然后切入到ls这个func，和fmtname这个func

  1. 任何阅读源码都应该从main函数开始 (不知道从哪里看来的，也有可能是我自己认为的)
    • ls.c的main写的特别简单 其实就是一个for循环，实则xv6的ls 和我们熟知的linux的ls是不太相同的，xv6的ls是可以接受多个目录作为参数的(由for循环可以看出) 如 ls a b 这个命令就会展示文件夹a和文件夹b里面的所有文件(include “.”,"..") 如果直接ls，则默认展示当前工作目录的所有文件(if 语句看出)
  2. 大头还是ls这个func， ls接受参数为路径path(可以是绝对路径或者是相对路径)，主要是对整体结构有大致的掌握
    • struct dirent de 这个指的是目录项这一结构体（在kernel/fs.h中定义），其实目录也是一种文件，里面就是存放了一系列的目录项(结构体，结构体的内容请看下面源码部分的注解)
    • struct stat st 这个指的是文件的统计信息（在kernel/stat.h中定义），包含文件类型（目录或文件）/inode/文件引用nlink/文件大小/存放fs的disk dev
    • open系统调用 打开文件，第二个参数指示的是打开方式，0代表的是O_RDONLY只读的形式。返回值是file descriptor >=0，<0说明open失败（fstat类似 不做赘述）
    • 下面进入最主要的switch 结构里面，这里比较简单只有两个case，一个是文件如何处理，一个是目录怎么处理
      • 如果是文件就很简单了，直接打印信息就成了 你会发现这里的st.type有三个值分别是({1:目录,2:文件,3:console}) fmtname返回值就是文件名称
      • 目录的操作就相对复杂很多
        • if语句只是一个检查缓存有没有溢出，没有什么其他意思
        • 下面的几句话就是在拼接字符串，目的是为了以路径名访问这个目录里面的内容， read操作就是在访问目录内容
        • while里面的if(de.inum==0)代表了此文件夹无文件，所以直接continue，（continue操作后进行下一次read ）
        • memmove函数原型是void* memmove(void *vdst,const void *vsrc,int n) 实现在（user/ulib.c）,主要意思就是内存之间的迁移，在ls.c里面的意思就是将de.name的内容覆盖到p指向的指针中
        • 最后最值得注意的其实应该是这个while循环 ：while(read(fd, &de, sizeof(de)) == sizeof(de)) ，每次read只是read一个de的大小（也就是一个目录项），只有read到最后一个目录项的下一次read才会返回0，也就不满足while循环条件退出循环，这是一点很值得注意的细节！ 如果对系统调用不了解的话很容易看不懂 ，建议阅读xv6 book或者我写的关于system call的详解，里面就有清晰的讲到这一点，就是read返回值以及offset的问题。
  3. fmtname func 这个函数就是返回字符串指针的
  • for 循环部分就是在从后往前遍历字符串，让p指向’/’,p++则是让p指向’/‘后的第一个字符
  • 其他全在下文源码分析的注解中 至此 本文结束!!!!! bingo，本文详细的分析了qemu里面的user/ls.c的源码，如果想做好find的话，这个过程必不可缺，所以加油吧! 下面是我注解的源码部分

#include "kernel/types.h"
#include "kernel/stat.h"
#include "user/user.h"
#include "kernel/fs.h"

char*
fmtname(char *path)
{
  static char buf[DIRSIZ+1];
  char *p;

  // Find first character after last slash.
  for(p=path+strlen(path); p >= path && *p != '/'; p--) //p initial advance len(path) and back
    ;
  p++; // if path_name = ./file1/a/b/file_c  then p-> "file_c"

  // Return blank-padded name.
  if(strlen(p) >= DIRSIZ)
    return p;//return file name
  memmove(buf, p, strlen(p)); //if len(p)<DIRSIZ, move *p content into buf,
  memset(buf+strlen(p), ' ', DIRSIZ-strlen(p)); //对buf[len(p)]开始的部分进行初始化为空，就是占满buf的空间
  return buf;
}

void
ls(char *path)
{
  char buf[512], *p;
  int fd;
  struct dirent de; //directory entry 目录项
  struct stat st;

  if((fd = open(path, 0)) < 0){
    fprintf(2, "ls: cannot open %s\n", path);
    return;
  }

  if(fstat(fd, &st) < 0){
    fprintf(2, "ls: cannot stat %s\n", path);
    close(fd);
    return;
  }

  switch(st.type){
  case T_FILE:
    printf("%s %d %d %l\n", fmtname(path), st.type, st.ino, st.size);
    break;

  case T_DIR:
    if(strlen(path) + 1 + DIRSIZ + 1 > sizeof buf){
      printf("ls: path too long\n");
      break;
    }
    strcpy(buf, path); //copy from path to buf
    p = buf+strlen(buf);// means p->buf[0] advance strlen(buf)--> p->buf[strlen(buf)]也就是现在p->buf[strlen(buf)]
    *p++ = '/';
    while(read(fd, &de, sizeof(de)) == sizeof(de)){
      if(de.inum == 0) //此文件夹无文件
         continue;
      memmove(p, de.name, DIRSIZ); //拼接路径字符串 like ./a/b + p
      p[DIRSIZ] = 0;
      if(stat(buf, &st) < 0){
        printf("ls: cannot stat %s\n", buf);
        continue;
      }
      printf("%s %d %d %d\n", fmtname(buf), st.type, st.ino, st.size);
    }
    break;
  }
  close(fd);
}

int
main(int argc, char *argv[])
{
  int i;

  if(argc < 2){
    ls(".");
    exit(0);
  }
  for(i=1; i<argc; i++)
    ls(argv[i]);
  exit(0);
}
