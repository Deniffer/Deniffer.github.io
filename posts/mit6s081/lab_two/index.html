lalala!!! 我来写lab two的syscall的记录啦，lab two我觉得相对于lab two来讲呢可能要更简单一点，它的目的在于让我们理解system call 到底是怎么work的，我们一般对于system call的了解就基本是(书面上)：某个应用程序需要执行一些它无法执行的指令(privilege instruction)，要通过system call的方式将cpu交给os，然后os通过检查用户程序是否拥有合理的权限来执行这个system call(如果指令非法就拒绝执行）。其实关键就在于isolation，因为要实现强isolation，所以用户程序不得直接操纵硬件（disk、memory等）所以os提供了某种抽象，比如可以理解（file其实是disk的抽象，file抽象了disk里面各个block，file这一层次的抽象，给用户可以直接通过对文件的操控来实现对硬盘的操作）因此，每当用户请求（open、read、write） file的时候，这些操作都是需要system call来执行的。 再扯一下，那什么是内存的抽象呢？其实exec（system call）就是内存的抽象， 我们知道每当我们fork 一个父进程的时候，子进程可以获得所有父进程的资源（包括file descriptor、stak、page table 等），如果只是这样的话，那么子进程其实就是父进程的一个duplicate，他将会执行父进程的所有指令！这并没有什么意义!真正让子进程有意义的地方，应该是它可以执行一些别的指令（父进程所不具有的），那问题的关键就在于如何让子进程执行别的指令？这时候exec就登场了，举个简单的例子，看下面的c程序

增加sysinfo 系统调用

  • 很多零零碎碎的我就不讲了，主要还是以全过程展示一下阅读源码的一些感悟
  • 首先是弄明白 filestat 这个function的整个流程， 这个函数接受两个参数，一个是struct file，一个是user virtual add,主要目的就是为了拷贝来自file的metadata到struct st上，
    • 解析：首先是*p指的是本进程，关于st的结构可以到(kernel/stat.h)中查看，就是一些文件的相关信息
    • 至于这里为什么要f->type进行if判断我也是没有什么想法 file的struct 定义在kernel/file.h, 可能是指明file怎么来的？我是这么想的，然后我们看一下f->ip的结构，f->ip应该是只有FD_INODE 和FD_DEVICE 才有的，一个指向inode的指针，这里的stati(f->ip,st)就是将file->ip(也即是struct inode)的信息 copy到 struct st中，保存了dev_num,inode,file_type,nlink(软、硬链接),file_size
    • 重点在于这个copyout function，定义在(kernel/vm.c)中，函数签名是*copyout(pagetable_t pagetable, uint64 dstva, char src, uint64 len) 对应到这里就是，p->tablepage就是当前进程的页表，addr就是virtual address 是一个指向struct stat的指针地址，st就是我们想要copy的source，sizeof(st)就是要copy的len， 要注意！这里的virtual address是在page table 里面的，为什么要传进page table这个参数，就是为了找到该virtual address 对应的physical address，然后将对应的信息传到这个指针里，也就是内存的某些地方(谨记：内存是physical addr 如0xffffff)。
    • 然后我们再来看sys_fstat(void)这个函数，最重要的地方是在if语句里面，如何理解这个if语句的问题，其中有两个函数argfd和argaddr， 首先我们来看argfd这个函数
    • argfd的注释 （Fetch the nth word-sized system call argument as a file descriptor，and return both the descriptor and the corresponding struct file.）其实也已经解释得很好了，但是我们想看看他到底做了些什么东西，函数原型是*argfd(int n, int *pfd, struct file *pf)
    • 层层递进，我们最终会进入argraw这个func里面，其实这个argraw的return value就是p->trapframe->a0，也就是我们所说的syscall的return value，然后返回到argint这个func，*ip这个int指针指向的就是上面的说的return value ，然后返回到argfd，然后将return value作为fd（file descritor）.

// Get metadata about file f.
// addr is a user virtual address, pointing to a struct stat.
int
filestat(struct file *f, uint64 addr) //good point!
{
  struct proc *p = myproc();
  struct stat st;
  
  if(f->type == FD_INODE || f->type == FD_DEVICE){
    ilock(f->ip);
    stati(f->ip, &st); //from fs.c  the main func is to copy info from f->ip to st
    iunlock(f->ip);
    if(copyout(p->pagetable, addr, (char *)&st, sizeof(st)) < 0) //addr stands for virtual addr ,st stands for src(Physical address)
      return -1;
    return 0;
  }
  return -1;
}
