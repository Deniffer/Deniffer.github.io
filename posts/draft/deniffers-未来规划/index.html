Linux Command Usage Analysis

  • Blod text : type exactly as shown
  • Italic text (underlined text) : replace with appreciate argument.
  • [-abc] : any or all arguments within are optional.
  • -a | -b : options delimited by | can not be used together
  • Argument … : argument is repeatable.
  • [expression] … : entire expression within is repeatable.

If an option that does not have option-arguments is repeated, the results are undefined, unless otherwise stated.

Ellipses ( "..." ) are used to denote that one or more occurrences of an operand are allowed. When an option or an operand followed by ellipses is enclosed in brackets, zero or more options or operands can be specified. The form:

# 以man top 为例 讲解man的使用 主要覆盖语法的部分
SYNOPSIS
     top [-a | -d | -e | -c mode]
         [-F | -f]
         [-h]
         [-i interval]
         [-l samples]
         [-ncols columns]
         [-o key | -O skey]
         [-R | -r]
         [-S]
         [-s delay-secs]
         [-n nprocs]
         [-stats keys]
         [-pid processid]
         [-user username]
         [-U username]
         [-u]
1. ｜ 在语法中代表着选项互斥 ， 
2. [-p option_argument] 代表该选项是可选的 不是必须的 并且出现该选项的时候一定有option_argument， 而[-p [option_argument]]代表选项参数是可选的
